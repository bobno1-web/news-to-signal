# 라이브 검증 체크리스트: 루프13 판단 4종(B-1~B-4)의 LLM 실거동 — API 키 확보 시 실행

- 발의: 검증방 / 루프13 검증 / 2026-07-20
- 성격: **무-API 미검 항목.** 루프13 B(주체성 결정성·감사중요 조임·사건군·자기모순)는 판단 프롬프트
  변경이라 LLM 실거동은 라이브(ANTHROPIC_API_KEY) 필요. 이 환경엔 키가 없어 결정론(코드·표시·스키마·
  파싱·접기·정합) 부분만 실증했다. 아래는 크레딧 확보 시 검증방이 돌릴 체크리스트다. 프롬프트 변경으로
  기존 판단 캐시는 전량 무효(새 프롬프트=새 캐시 키)라 라이브 재실행은 실비용이다.
- 실증 완료(무-API): 프롬프트 .format 무오류·B 블록 삽입, event_group 스키마 왕복·하위호환·덤프,
  사건군 접기 정합(증발0·대표=심각도최상위·오병합 가드·병합 아님), 폴백 드롭0(5사), 전체 138 pass.

## 라이브로만 확인 가능한 항목 (억지 통과 주장 금지)

### 1. [B-1] 서머리·묶음형 기사가 실제로 '타사언급(C)'으로 판정되는가 — 결정성 포함
- 대상: "삼성전자 2Q 영업익…" 주간 브리핑 속 CJ 과징금 한 줄(루프10 발단 재현), 기업기상도·주간
  묶음형 기사.
- 확인: (a) C 로 다는가, (b) subject_role_reason 에 '왜 C/왜 B'가 적히는가(A/B/C 전부 필수화 검증),
  (c) ★같은 기사 반복 실행에서 C↔상단 뒤집힘이 사라졌는가(결정성 — B-1 의 핵심 동기).
- 가드: '건설사 부채비율…이 회사 489%'(복수주체)·지분46% 자회사(제목에 모회사명 없음)는 여전히 B 로
  살아있는가(놓침 방지 — C 로 과잉강등되면 실패).

### 2. [B-2] 감사중요(audit_attention) true 비율이 실제로 내려가는가
- 기준선: 루프11 라이브 CJ 표시 20건 중 14건(70%) true(남발).
- 확인: 루프13 프롬프트('세 관문' 전부 통과 요구)로 재실행 시 true 비율이 얼마로 내려가는가(수치 기록).
  ★2300억 세일앤리스백은 여전히 true(관문 통과)로 유지되는가, 정책 검토단계·업계동향·홍보/애널의견은
  false 로 빠지는가.
- 주의: '70%→N%'의 N 은 라이브로만 얻는다. 임계치·건수상한이 코드에 없음은 무-API 로 확인(검증4 통과).

### 3. [B-3] 담합 9건이 실제로 '하나의 event_group' 라벨로 묶이는가
- 기준선: 루프 초 CJ 표시 20건 중 9건이 제분·설탕 담합 한 사안(도배).
- 확인: LLM 이 그 9건에 '같은 라벨'을 다는가(국면 심의/확정/소송 달라도), 서로 다른 사안(세무조사·자회사
  청산·세일앤리스백)에 '다른 라벨'을 다는가, 애매한 것은 빈 값으로 두는가(오병합=증발 금지).
- 표시 결과: 대표 1건만 상단, 나머지 8건이 그 아래 접힘(펼치면 전부). 접기 메커니즘·정합은 무-API 로
  이미 실증 — 남은 건 'LLM 이 실제로 같은 라벨을 매기는가'뿐.

### 4. [B-4] 스스로 '경미·소액'이라 판단한 건이 상단에서 빠지는가
- 확인: LLM 이 one_line_reason·이유에 '경미/소규모/소액/국지적'이라 쓴 건의 severity_rank 가 실제로
  낮아져 화면 하단으로 가는가(판단층 자기일관성). ★'미상≠작음'은 유지되는가(금액 미상 본질위험 사건이
  '경미'로 낮아지면 실패). ★표시층은 '경미' 문자열을 필터링하지 않음(무-API 확인, 검증4) — 낮춤은
  판단층이 해야 화면에 반영된다.
