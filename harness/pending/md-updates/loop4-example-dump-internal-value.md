## 처방 (반영 완료 — 루프5, 후보 (b) 채택)
`serialize.full_dump` 이 예시를 심각도순으로 정렬해 저장하고 severity_rank 를 '위치 서수'
(N..1)로 덮어쓴다. load_full→rank_by_severity 가 저장 순서를 그대로 복원하므로 예시 정렬
재현은 유지되고, 커밋되는 파일(out/results/*.json)에는 모델의 내부 홀리스틱 원값이 남지
않는다. 실측: 두 예시 파일의 severity_rank 가 {1..N} 순열(위치 서수)로 재저장됨. 브라우저
payload 는 애초에 내부값 부재(judgment_view 가 제외) — 이는 그대로 유지.

## 관측(별건, 매우 낮음 — 잔류, 이번 스코프 아님)
- server 의 `_KEYS`·`_RESULTS` 는 모듈 전역이라 Flask 기본 threaded 에서 동시요청 경합
  여지가 있다. 로컬 단일 사용자(127.0.0.1) 도구라 실害는 낮으나, 다중 사용자로 확장 시
  세션 격리가 필요하다. 판단 왜곡과 무관. (표시 스코프 밖 — 다음 판단/구조 루프에서 검토)
