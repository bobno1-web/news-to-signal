# 본문 크롤링 (article-fulltext.md) — 유예된 확장

> 상태: 지금 구현하지 않는다. 현재 판단 재료는 제목 + 스니펫(요약)뿐이다.

## 현재 상태
네이버 뉴스 검색 API는 제목·링크·짧은 요약(description)을 준다. 본문 전체는
주지 않는다. 현재 시스템은 이 얇은 재료(제목+스니펫)로 저확신 1차 판단만
한다.

## 확장 시 (미래)
링크를 따라가 본문을 가져오면 판단 품질이 오른다. 특히 계정 연결과 규모
판단의 정밀도가 올라간다.

## 왜 지금 안 하나
- 사이트마다 크롤링이 실패하고, 이용약관·차단 문제가 붙는다.
- 하네스 원칙: 얇은 재료로 하는 1차 판단이 값을 증명한 다음에 붙일 무게다.
- 본문이 없어 판단이 "일반 리스크, 계정 미정"으로 기권하는 것은 버그가
  아니라 정직함이다(원칙 4).
