# 기업 위험 뉴스 모니터링

> 기업 뉴스에서 재무·감사 위험 신호를 찾아, 심각도순으로 정렬하고
> 담당 계정별로 매핑해 근거와 함께 보여줍니다. — **판정은 감사인이 합니다.**

---

## 이런 도구입니다

회사명을 넣으면 외부 **뉴스**를 중립적으로 수집하고, 각 뉴스를 **이해**해서
기업의 **재무·감사 위험 신호**를 **심각도순**으로 정렬한 뒤, **계정 담당자별로**
매핑해 보여줍니다. 모든 판단에는 **원문 근거 문장**이 함께 붙습니다.

이 도구는 위험을 **판정하지 않습니다.** 키워드로 분류하지 않고, 뉴스를 이해해서
"결론"이 아니라 **"가설 + 근거"**를 제시합니다. 근거를 못 대면 그 판단은 버리고,
재료가 부족하면 **기권**합니다. 최종 판정은 화면의 근거를 읽은 감사인이 합니다.

## 누구에게 · 왜

감사에서 **놓침(거짓음성)은 오탐보다 치명적**입니다. 하지만 뉴스는 많고, 재무제표에
실제로 영향을 주는 신호는 감정적 헤드라인에 묻히기 쉽습니다. 이 도구는 감사인이
그런 신호를 **놓치지 않도록** 뉴스를 이해해 위로 올리고, "왜 위험한지"를 5초 안에
검증할 수 있는 **근거와 함께** 내놓습니다. 아무 뉴스도 시야에서 완전히 지우지
않으며(낮은 우선순위는 접되 항상 접근 가능), **계정 미배정 뉴스도 숨기지 않습니다.**

## 이렇게 나옵니다

**심각도순 보기(기본)** — 재무제표 영향이 큰 뉴스가 위로. 각 뉴스는 3층으로 열립니다.

- **1층(항상):** 제목 · 방향(악재/호재/중립) · 규모(비율 또는 "규모 미상") · 계정 태그(가설) · 한 줄 이유
- **2층(펼침):** 축별 근거 · **원문 근거 문장** · 규모 근거 · 계정 가설의 근거
- 내부 점수는 화면에 표시하지 않습니다 — 순위는 '위치'로, 판단은 '말'로 드러냅니다.

`[스크린샷: 심각도순 보기 — 1층 요약과 펼친 2층 근거]`

**계정별 보기** — 세 바구니로 라우팅합니다: ① 내 계정 / ② 계정 미배정(공용 큐) / ③ 남의 계정.
바구니 안에서만 심각도로 정렬하며, **미배정 바구니는 숨기지 않습니다**(못 붙였다는 건 사람이 봐야 한다는 뜻).

`[스크린샷: 계정별 보기 — 세 바구니]`

**내 계정 선택** — 그 회사가 DART에 **실제로 가진 계정**만 후보로 뜹니다(임의 계정 없음).
담당 계정을 고르면 거기 걸리는 뉴스가 "내 계정" 바구니로 모입니다. **계정 선택은 '표시 렌즈'일 뿐**,
어떤 뉴스의 심각도·방향·근거도 바뀌지 않습니다.

`[스크린샷: 내 계정 선택]`

## 실행 방법 (원클릭)

1. 필요한 패키지 설치(최초 1회):
   ```
   pip install -r requirements.txt
   ```
2. 실행:
   - **Windows:** `start_news_risk_monitor.bat` 더블클릭
   - **macOS/Linux:** `chmod +x start_news_risk_monitor.sh` 후 `./start_news_risk_monitor.sh`
3. 브라우저가 자동으로 열립니다(안 열리면 http://127.0.0.1:8765 ).

**API 키 (화면에서 입력)** — 화면 ①에서 아래 키를 입력합니다. 어디서 받는지는 `.env.example` 참고.

| 키 | 용도 |
|---|---|
| `NAVER_CLIENT_ID` / `NAVER_CLIENT_SECRET` | 네이버 뉴스 수집 |
| `ANTHROPIC_API_KEY` | 뉴스 판단(LLM) |
| `OPENDART_API_KEY` | 회사 계정과목 목록 + 재무 규모(분모) |

> **키는 서버 메모리에만** 저장됩니다. 파일·로그에 남지 않고, 서버를 끄면 사라집니다.
> 키가 없거나 틀리면 임의값으로 채우지 않고 **명확한 오류로 멈춥니다**(억지 결과 없음).

**키 없이 결과 보기(예시)** — 키 발급 전이라면 화면 ①의 **"키 없이 예시 결과 보기"**로
미리 돌려둔 실제 실행 결과를 확인할 수 있습니다(`out/results/`). 심각도순·계정별 보기·계정 선택까지 동일하게 작동합니다.

## 이 도구가 신경 쓴 것

1. **키워드가 아니라 이해.** 사건을 고정 목록으로 분류하거나 단어 등장으로 심각도를 켜지
   않습니다. LLM이 뉴스를 이해해 판단하므로, 목록에 없던 새 회사·새 사건 유형에도 작동합니다.
2. **심각도는 감정이 아니라 재무제표.** "CEO 막말"(감정적 악재, 재무 무관)보다 "핵심 계약
   종료"(밋밋하나 매출 직격)를 위에 둡니다. 심각도는 여러 축으로 **다각도** 판단하되,
   가중치를 곱하는 점수 공식이 아니라 판단을 드러내는 창문으로 씁니다.
3. **결론이 아니라 근거로 검증.** 모든 판단에 원문 근거 문장이 붙습니다. 화면에 나오는 것은
   내부 숫자가 아니라 사람이 5초에 검증 가능한 이유이고, 규모·계정은 확실할 때만 제시하고
   불명이면 기권합니다. 그래서 감사인이 스스로 바로잡을 수 있습니다.
