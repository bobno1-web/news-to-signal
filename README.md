# 기업 위험 뉴스 모니터링

> 기업 뉴스에서 재무·감사 위험 신호를 찾아, 심각도순으로 정렬하고
> 담당 계정별로 매핑해 근거와 함께 보여줍니다. — **판정은 감사인이 합니다.**

---

## 이런 도구입니다

감사인은 담당 회사의 외부 뉴스에서 재무·감사에 영향을 줄 위험 신호를 놓치면
안 됩니다. 그런데 뉴스는 방대하고 대부분은 무관하거나 홍보성입니다. 기존 뉴스
알림은 "키워드가 들어간 기사"를 던져줄 뿐, 그것이 어느 계정에 영향을 주는 재무
신호인지, 감사 관점에서 얼마나 심각한지는 판단해주지 않습니다.

이 도구는 회사 이름 하나만 넣으면 —

1. 그 회사의 최근 1년치 뉴스를 **넓게, 중립적으로** 수집하고
2. 각 뉴스가 재무·감사에 닿는 위험 신호인지 **이해로 판단**하고
3. 심각도순으로 정렬해, 담당 **계정별로 나눠** 보여주고
4. 각 판단마다 **근거 문장**을 함께 제시합니다

감사인은 흩어진 뉴스를 직접 훑는 대신, 위험 신호가 정리된 결과를 바로 확인하고
스스로 판정할 수 있습니다.

---

## 누구에게 · 왜 필요한가

**대상: 회계법인 감사팀 (계속감사 모니터링 · 위험 평가 · 감사계획 수립)**

**감사인이 겪는 어려움**
담당 회사에 재무·감사상 문제가 생기고 있는지 외부 뉴스로 파악하려면, 수많은
기사를 직접 훑어야 합니다. 대부분은 홍보·단순 동향이라, 정작 중요한 신호
(워크아웃, 우발채무, 특수관계자 거래 등)가 그 사이에 묻힙니다.

**이 도구의 해결 방법**
뉴스를 넓게 수집한 뒤, 재무·감사에 닿는 악재와 위험 해소 신호만 골라 심각도순
으로 정렬합니다. 각 뉴스가 어느 재무제표 계정에 영향을 줄 수 있는지 가설을
붙이고, 그 근거가 된 문장을 함께 보여줍니다. 감사인은 정리된 결과를 확인하고,
근거를 보며 스스로 판정합니다.

**이렇게 쓸 수 있습니다**
- 계속감사 중, 담당 회사의 재무·감사 관련 후속 사건 모니터링
- 감사 위험 평가 시, 외부에서 관측되는 위험 신호 파악
- 감사계획 수립 전, 회사와 산업의 최근 이슈 확인

## 이렇게 나옵니다

**1. 결과 화면 — 심각도순으로 정렬된 위험 신호**
*(예: 태영건설 → 워크아웃·PF·부채비율 등 재무·감사 위험 뉴스)*

`[스크린샷: 결과 화면]`

**2. 근거 확인 — 각 판단의 근거 문장과 계정 가설**
*(왜 이게 악재인지, 왜 이 계정에 닿는지, 원문 근거를 펼쳐 확인)*

`[스크린샷: 카드 펼친 화면]`

**3. 계정별 보기 — 담당 계정으로 뉴스 라우팅**
*(재무상태표 표에서 담당 계정을 고르면, 그 계정에 걸린 뉴스가 모임)*

`[스크린샷: 계정 선택 화면]`

## 실행 방법

이 도구는 **내 PC에서 실행되는 로컬 도구**입니다.

1. 최초 1회: `pip install -r requirements.txt`
2. `start_news_risk_monitor.bat` 더블클릭 (Mac/Linux는 `.sh`)
3. 실행 창이 뜨고 브라우저가 자동으로 열립니다 (`http://127.0.0.1:8765/`)
4. 화면에서 API 키 4개를 입력하고, 회사명을 넣으면 시작됩니다

> 사용하는 동안 실행 창을 닫지 마세요 — 이 창이 서버를 유지합니다.
> 다 쓴 뒤 창을 닫으면 종료됩니다.

**필요한 API 키 4개**
- **Open DART** (무료) — 회사 재무·계정 정보
- **네이버 검색 API** (무료, Client ID / Secret) — 뉴스 수집
- **Anthropic (Claude) API** (사용량 기반 유료, 소액) — 뉴스 판단

입력한 키는 이 세션에만 쓰이고 파일·로그에 저장되지 않습니다. 서버를 끄면
사라집니다. 발급처 링크는 `.env.example`을 참고하세요.

> 직접 실행하지 않아도, 미리 돌려둔 예시 결과를 `out/results` 폴더에서
> 바로 확인할 수 있습니다.

## 이 도구가 신경 쓴 것

만들면서 가장 공들인 부분은 **"감사인이 신뢰하고 쓸 수 있는가"**였습니다.

**1. 키워드가 아니라 이해로 판단합니다**
"횡령·소송" 같은 부정 키워드 목록으로 위험을 잡으면, 목록에 없는 새 유형은
놓치고 업종마다 다른 위험의 언어를 담을 수 없습니다. 그래서 뉴스를 넓게
수집한 뒤, 각 기사가 재무·감사에 닿는 신호인지 **기사의 의미를 이해해서**
판단합니다. 고정된 키워드·점수표를 쓰지 않습니다.

**2. 위험을 여러 각도에서 따져 정렬합니다**
같은 악재라도 감사인에게 급한 정도는 다릅니다. 이 도구는 각 뉴스를 관련성
(정말 이 회사 일인가), 규모(회사 재무 대비 얼마나 큰가), 확실성(확정된 사실인가
가능성인가), 시점(과거인가 진행 중인가), 방향(악화인가 해소인가) 등 여러 각도에서
따져 심각도를 매기고, 그 순서로 정렬합니다. 단, 이 각도들을 점수 공식으로 굳히지
않고 "무엇을 먼저 물어야 하는가"의 관점으로만 씁니다 — 사건마다 어느 각도가
중요한지 다르기 때문입니다.

**3. 판단의 근거를 함께 붙여, 감사인이 스스로 검증할 수 있게 합니다**
각 판단 아래에는 그 판단이 근거한 **기사 원문 문장**이 함께 붙습니다. "왜
악재인지", "왜 이 계정에 닿는지"마다 실제 문장이 달려 있어, 감사인은 기사
전문을 일일이 열지 않고도 그 근거만으로 **"이 추론이 타당한가"를 바로 판단**할
수 있습니다. 근거 문장이 없는 판단은 애초에 화면에 올리지 않습니다. 이 도구는
위험을 확정하지 않으며, 최종 판정은 언제나 감사인의 몫입니다.

또한 이 도구는 특정 회사에 맞춘 것이 아니라, **어느 상장사든** 회사 이름만
넣으면 동작하도록 만들었습니다. (건설·조선·반도체·자동차·식품 등 여러 업종에서
확인)

---

*규모(금액 대비 비율)는 회사의 실제 재무 대비로 계산하되, 뉴스에 금액이 명확할
때만 계산하고 모호하면 "규모 미상"으로 남깁니다. 계정 정보는 DART 표준재무제표
기준입니다.*
